Creating a Webpage

A Summary Guide for Stage 1

Creating Visually Appealing Webpages

To create a webpage that looks visually striking, we would need to use at least two langauges, HTML and CSS.

HTML (Hypertext Markup Language) is used to define the structure and content of a webpage, and is the standard language used to create webpages. HTML is what makes the web "web-like", because with it we can control what is seen on a webpage, what it looks like, as well as add content (such as images, videos, and documents) and links to other webpages.

CSS (Cascading Style Sheets) helps us style a webpage. It is a language we use for choosing the appearance and formatting of our webpage; it allows us to easily control things like fonts, backgrounds, colors, borders, positions of elements, etc.

HTML Crash Course

When we code in HTML, we have to use a special syntax and follow specific rules so that our markup (code) can be converted to a webpage we desire. To do this, we use tags -- the basic HTML commands. When we think of HTML as a language, then we can think of tags as words. This is what a tag looks like:

<...>example</...>

The first set of brackets is the opening tag, and the second, with the forward slash "/", is the closing tag. We use tags to form elements, which are the actual content of our webpage. This is what an element looks like:

<b>text</b>
<em>text</em>

The first tag (the "b" tag), makes the text inside it appear bolded; the second ("em") makes the text inside it appear italicized on our webpage, as shown in this paragraph. Each of the two is an element.

Note that the green tag is an opening tag; the red tag is an ending tag. Between these two, we have content.

HTML tags can be nested -- one tag can be inside another. This is an example of a nested tag:

<b>this text is just bold <em> and this text is both bold and italicized</em></b>

The output with the nested tags above would look like this:
this text is just bold and this text is both bold and italicized

TIP
Do not forget to close a tag! Everthing following the opening tag is part of that tag.

A more complex tag includes attributes and values, and linking to other webpages or incorporaring images into our webpage are examples. Consider the following example:

<img src="www.webpage.com/image.jpg">

The value is the link to the page. The attribute is "src". Also note that in this example no closing tag is needed, because the image tag is an example of a void tag.

Another example:

<a href="www.webpage.com">Link to another webpage</a>

This tag links another webpage to our page, and the text "Link to another webpage" will appear as a clickable link to that webpage.

CSS Crash Course

With CSS, we can define the styles for our webpage. We can establish a style for a single element, every element of a specific type, or a group of identical elements that we have marked.

Incorporating a Style

To use a CSS style, we need to somehow incorporate it into our HTML code. There are three ways of doing so:

  1. In our HTML, we add a link to a separate file that contains our CSS code. Example:
    <link rel="stylesheet" type="text/css" href="style.css">
    Here, we are linking to a document (file) named style.css.
  2. We include a stylesheet in our HTML directly (this is typically done in the <head> element. Example:
    <style>div {color:blue}</style>
  3. We include an inline style directly in the HTML elements. Example:
    <div style="color:blue">this text would be blue</div>

How we set up a style matters; in the above order, each successive selection overrides the preceding style definition -- in other words, a style defined in a linked CSS file overrides the default browser style, whereas a stylesheet directly in our HTML code overrides the style defined in a linked CSS file, and an inline style in the element, in turn, overrides a stylesheet embedded in HTML.

TIP
Inline CSS as shown above is not recommended as our standard approach to adding style, because if we wanted to make a quick change to our style on the entire webpage, we would have to individually edit each element where we defined a style.
Writing CSS Code

To set a style for all elements of a particular type, we simply establish a style in our CSS code that applies to all elements of the type. We simply reference the HTML tag.

body {
    font-family: Helvetica
}

In the above example, everything that appears in the body of our webpage will use the font "Helvetica."

To set a style for elements that we have specified, we use a class. A class is a kind of a grouping of elements that is assigned a style of our choosing.

.lesson {
    font-size: 30px;
    color: #294860;
}

In the above example, the class "lesson" (which is defined in our HTML) will use a font size of 30 pixels, and the specific color we've defined in hex.

To set a style for a specific, single element, we use the id selector -- in other words, we assign a unique marker to just that particular element we want to change, and then write CSS code to change it.

#special {
    text-align: center;
    color: red;
}

In this example, the id "special" (which is defined - or, we can say, assigned to a particular element - in our HTML) will align the text in the center and use red color.

There are many things we can change about our webpage's appearance using CSS. These webpages are good references to have when defining CSS styles:

  • Mozilla Foundation CSS Reference
  • W3Schools.com CSS Tutorial
TIP
We can write comments in CSS that the computer won't "read":
/* this is a CSS comment */
Arranging Things On Our Website

By default, everything (all text, every element) we add to our webpage appears right next to the right of whatever came before it, without spacing inbetween. This means that we have to take steps to manage how we want our content to appear on the webpage.

A simple way to organize our webpage is by simply using tags to create whitespace -- an area of spacing between our elements so that our webpage does not appear disorganized or cluttered.

When writing text, we can start a new line using the BR tag. Example:

text text text <br>text text text <br>text text text

In this example, the text output actually displays like this on our webpage:
text text text
text text text
text text text

We can also organize text using the paragraph P tag; everything inside the tag will appear on our webpage as a distinct paragraph of text.

<p>text text text text text text text text text</p>

Another thing we can do is use "container" elements, such as div and span. These help us control how things are laid out on our webpage. Some HTML elements are inline while others are block -- what is means is that they behave differently on a webpage in terms of layout: whereas block elements form an invisible box around the things inside it, while an inline element has no such invisible border and takes only as much space as necessary.

Examples of inline elements:

<a>, <span>, <br>, <strong>, <em>, <b>

Examples of block elements:

<p>, <div>, <form>
Think Inside the Box

Picturing webpages as a collage of boxes makes it easier to understand and design them. When we conceptualize a design of a webpage, we can and should divide it using rectangular borders -- a practice called boxifying.

When we boxify, we get clues about what the different elements are that we will need to put on our webpage. So, working from our sketch, we start thinking about how the webpage will be coded and can begin outlining sections in our HTML.

<html>
<head>
</head>
<body>
   <h1> </h1>
   <div>
      <div>
      </div>
      <div>
      </div>
   <div>
      <div>
         <div>
         </div>
      </div>
   </div>
   </div>
</body>
</html>

In a way, we work backwards by putting together our code based on a conception of what our website should look like.

Sizing and Arranging Content

Above, we made a case for thinking of all website content as boxes. To organize our webpage the way we want to, we will need to be mindful how we size and arrange these boxes. The first thing that we will need to remember is that the width and height of our box does not equal the actual dimensions of the content itself -- there are also the margin, border, and padding to consider. This concept is well illustrated [LINK] here.

TIP
Actual element width =
left border + left padding + content + right padding + right border

There's a way to automatically perform this calculation in CSS so that we do not have to worry about it:

*{
    -webkit-box-sizing: border-box;
    -moz-box-sizing: border-box
    -ms-box-sizing: border-box;
    box-sizing: border-box;
}

We can set box sizes in pixels or as a percentage (screen area); in the case of the former, size of our element is fixed irrespetive of the screen size on which it is displayed, and in the case of percentages, the output will self-adjust depending on the size of the screen (browser window size).

In CSS, there is a convenient way that allows us to place boxes next to one another -- it is called the Flex Box.

.app {
    display: -webkit-flex;
    display: flex;
}
